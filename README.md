# minutes-bot — этап 1: сбор сообщений группы

Скелет Telegram-бота на aiogram 3 + PostgreSQL для безмолвного сбора всех сообщений группы. На этом корпусе позже строится «секретарь» с напоминаниями.

## Что внутри

```
bot/
  main.py        — aiohttp-приложение + webhook
  config.py      — настройки из env
  db.py          — пул asyncpg, схема, запись
  handlers.py    — catch-all + /ping + /stats
requirements.txt
render.yaml      — Blueprint для Render (web + Postgres)
.env.example
```

## Развёртывание

**1. BotFather.** В `@BotFather`: `/setprivacy` → бот → **Disable**. Затем `/setjoingroups` → **Enable**, если ещё не. Если бот уже сидит в группе — **удалите и добавьте заново**, иначе новые права не подхватятся. Дайте ему права администратора (минимально — без специальных прав; галочка «может читать все сообщения» появится автоматически после Disable privacy).

**2. GitHub.** Создайте приватный репозиторий, залейте содержимое этой папки.

**3. Render → New → Blueprint** → укажите репо. Render прочитает `render.yaml`, создаст web service и Postgres.

**4. Переменные.** В Dashboard сервиса задайте:
- `BOT_TOKEN` — токен от BotFather
- `WEBHOOK_BASE` — `https://<имя-вашего-сервиса>.onrender.com` (виден после первого деплоя)

`WEBHOOK_SECRET` Render сгенерирует сам, `DATABASE_URL` подставится из БД.

**5. Redeploy.** После того, как `WEBHOOK_BASE` задан, перезапустите сервис — на старте бот сам вызовет `setWebhook`.

**6. Проверка.** В группе `/ping` → бот ответит `pong`. Напишите 2–3 произвольных сообщения, потом `/stats` — должно показать счётчик.

## Локальный запуск (для отладки)

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
# поднимите локальный Postgres, экспортируйте env
export BOT_TOKEN=... DATABASE_URL=... WEBHOOK_BASE=https://xxx.ngrok.io WEBHOOK_SECRET=...
python -m bot.main
# в другом терминале: ngrok http 10000 → URL подставьте в WEBHOOK_BASE
```

## Что дальше (этап 2)

Поверх таблицы `messages` навешивается:
- периодическая job (APScheduler), которая берёт новые сообщения и отдаёт в LLM с промптом «выдели поручения»;
- таблицы `commitments` / `reminders`;
- команды `/tasks`, `/done <id>`, `/digest`;
- транскрипция голосовых через Whisper при необходимости.

Этап 2 сознательно не включён в этот скелет — сначала нужен корпус, на котором можно отладить извлечение задач.

## Подводные камни

- Если переключите privacy mode после добавления — **переподключите бота к группе**.
- При повышении группы до супергруппы `chat_id` меняется (становится отрицательным `-100...`). Старые записи в БД останутся со старым chat_id — учтите при выборках.
- Free-tier Postgres у Render живёт 90 дней — после этого нужен платный или миграция.
- Telegram держит `file_id` сколько угодно, но он привязан к боту: с другого токена скачать файл не получится.
- Уведомите участников группы, что ведётся запись и автоматическая обработка — это базовый комплаенс.
